= 계정 효율성 얻기
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
사용할 수 있습니다 `GetAccountEfficiency` 볼륨 계정에 대한 효율성 통계를 얻으려면.  이 방법은 매개변수로 지정한 계정에 대해서만 효율성 정보를 반환합니다.



== 매개변수

이 방법에는 다음과 같은 입력 매개변수가 있습니다.

|===
| 이름 | 설명 | 유형 | 기본값 | 필수의 


 a| 
계정ID
 a| 
효율성 통계가 반환되는 볼륨 계정을 지정합니다.
 a| 
정수
 a| 
None
 a| 
예

|===


== 반환 값

이 메서드는 다음과 같은 반환 값을 갖습니다.

|===
| 이름 | 설명 | 유형 


 a| 
압축
 a| 
계정의 모든 볼륨에 대한 데이터 압축을 통해 절약된 공간의 양입니다.  "1" 값은 데이터가 압축되지 않고 저장되었음을 의미하는 비율로 표시됩니다.
 a| 
뜨다



 a| 
중복 제거
 a| 
계정의 모든 볼륨에 대한 데이터를 복제하지 않음으로써 절약된 공간의 양입니다.  비율로 표시됩니다.
 a| 
뜨다



 a| 
누락된 볼륨
 a| 
효율성 데이터를 쿼리할 수 없는 볼륨입니다.  볼륨 누락은 가비지 수집(GC) 주기가 1시간 미만이거나, 네트워크 연결이 일시적으로 끊어졌거나, GC 주기 이후에 서비스가 다시 시작된 경우에 발생할 수 있습니다.
 a| 
정수 배열



 a| 
씬프로비저닝
 a| 
데이터를 저장하기 위해 할당된 공간의 양에 대한 사용된 공간의 비율입니다.  비율로 표시됩니다.
 a| 
뜨다



 a| 
타임스탬프
 a| 
가비지 수집(GC) 후 효율성 데이터가 마지막으로 수집된 시간(UTC+0 형식)입니다.
 a| 
ISO 8601 날짜 문자열

|===


== 요청 예시

이 방법에 대한 요청은 다음 예와 유사합니다.

[listing]
----
{
   "method": "GetAccountEfficiency",
   "params": {
     "accountID": 3
   },
   "id": 1
}
----

이 메서드는 다음 예와 유사한 응답을 반환합니다.

[listing]
----
{
   "id": 1,
   "result": {
       "compression": 2.020468042933262,
       "deduplication": 2.042488619119879,
       "missingVolumes": [],
       "thinProvisioning": 1.010087163391013,
       "timestamp": "2014-03-10T14:06:02Z"
   }
}
----
