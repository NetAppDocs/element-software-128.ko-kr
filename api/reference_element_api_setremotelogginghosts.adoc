= 원격로깅호스트 설정
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
당신은 사용할 수 있습니다 `SetRemoteLoggingHosts` 스토리지 클러스터의 노드에서 중앙 로그 서버로 원격 로깅을 구성하는 방법입니다.  원격 로깅은 기본 포트 514를 사용하여 TCP를 통해 수행됩니다.  이 API는 기존 로깅 호스트에 추가되지 않습니다.  오히려 현재 존재하는 값을 이 API 메서드에서 지정한 새 값으로 대체합니다.  사용할 수 있습니다 `GetRemoteLoggingHosts` 현재 로깅 호스트가 무엇인지 확인한 다음 사용하세요. `SetRemoteLoggingHosts` 현재 및 새로운 로깅 호스트의 원하는 목록을 설정합니다.

 a|

 a| 
예

|===


== 반환 값

이 메서드에는 반환 값이 없습니다.



== 요청 예시

이 방법에 대한 요청은 다음 예와 유사합니다.

[listing]
----
{
   "id": 1,
   "method": "SetRemoteLoggingHosts",
   "params": {
     "remoteHosts": [
        {
          "host": "172.16.1.20",
          "port": 10514
        },
        {
          "host": "172.16.1.25"
        }
      ]
   }
}
----
