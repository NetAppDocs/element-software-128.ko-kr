= 요청 엔드포인트
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
API에서 사용되는 요청 엔드포인트에는 세 가지 유형(스토리지 클러스터, 스토리지 클러스터 생성, 노드별)이 있습니다.  항상 Element 소프트웨어 버전에서 지원하는 최신 엔드포인트를 사용해야 합니다.

API의 세 가지 요청 엔드포인트는 다음과 같은 방식으로 지정됩니다.



== 클러스터 API 메서드

스토리지 클러스터 전체 API 요청에 대한 HTTPS 엔드포인트는 다음과 같습니다. `+https://<mvip>/json-rpc/<api-version>+` , 어디:

* `<mvip>`스토리지 클러스터의 관리 가상 IP 주소입니다.
* `<api-version>`는 사용 중인 API 버전입니다.




== 클러스터 생성 및 부트스트랩 API 메서드

스토리지 클러스터를 생성하고 부트스트랩 API 요청에 액세스하기 위한 HTTPS 엔드포인트는 다음과 같습니다. `+https://<nodeIP>/json-rpc/<api-version>+` , 어디:

* `<nodeIP>`클러스터에 추가하는 노드의 IP 주소입니다.
* `<api-version>`는 사용 중인 API 버전입니다.




== 노드별 API 메서드

개별 스토리지 노드 API 요청에 대한 HTTPS 엔드포인트는 다음과 같습니다. `+https://<nodeIP>:442/json-rpc/<api-version>+` , 어디:

* `<nodeIP>`는 스토리지 노드의 관리 IP 주소이고, 442는 HTTPS 서버가 실행 중인 포트입니다.
* `<api-version>`는 사용 중인 API 버전입니다.
