= GetNodeFipsDrivesReport
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
당신은 사용할 수 있습니다 `GetNodeFipsDrivesReport` 스토리지 클러스터의 단일 노드에 대한 FIPS 140-2 드라이브 암호화 기능 상태를 확인하는 방법입니다.  이 방법은 개별 스토리지 노드에 대해 실행해야 합니다.



== 매개변수

이 방법에는 입력 매개변수가 없습니다.



== 반환 값

이 메서드는 다음과 같은 반환 값을 갖습니다.

|===
| 이름 | 설명 | 유형 


 a| 
fips드라이브
 a| 
이 노드에 대한 FIPS 140-2 기능 지원 상태를 포함하는 JSON 개체입니다. 가능한 값:

* 없음: 노드가 FIPS를 지원하지 않습니다.
* 부분적: 노드는 FIPS를 지원하지만 노드의 모든 드라이브가 FIPS 드라이브는 아닙니다.
* 준비: 노드가 FIPS 기능을 갖추고 있으며 노드의 모든 드라이브가 FIPS 드라이브입니다(또는 드라이브가 전혀 없습니다).

 a| 
끈

|===


== 요청 예시

이 방법에 대한 요청은 다음 예와 유사합니다.

[listing]
----
{
  "method": "GetNodeFipsDrivesReport",
  "params": {},
  "id" : 1
}
----


== 응답 예시

이 메서드는 다음 예와 유사한 응답을 반환합니다.

[listing]
----
{
    "id": 1,
    "result": {
        "fipsDrives": "None"
    }
}
----
