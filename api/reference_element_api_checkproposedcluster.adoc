= 제안된 클러스터 확인
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
당신은 사용할 수 있습니다 `CheckProposedCluster` 2노드 스토리지 클러스터에서 지원되지 않는 노드 유형이나 혼합 노드 기능의 불균형과 같이 시도에서 발생할 수 있는 오류나 오류를 식별하기 위해 스토리지 클러스터를 생성하기 전에 스토리지 노드 세트를 테스트하는 방법입니다.



== 매개변수

이 메서드에는 다음과 같은 입력 매개변수가 있습니다.

|===
| 이름 | 설명 | 유형 | 기본값 | 필수의 


 a| 
노드
 a| 
스토리지 클러스터를 구성하는 초기 스토리지 노드 세트의 스토리지 IP 주소 목록입니다.
 a| 
문자열 배열
 a| 
None
 a| 
예



 a| 
힘
 a| 
스토리지 클러스터의 모든 스토리지 노드에서 실행하려면 true로 설정합니다.
 a| 
부울
 a| 
None
 a| 
아니요

|===


== 반환 값

이 메서드는 다음과 같은 반환 값을 갖습니다.

|===


| 이름 | 설명 | 유형 


 a| 
제안된 클러스터 유효
 a| 
제안된 스토리지 노드가 유효한 스토리지 클러스터를 구성하는지 여부를 나타냅니다. 가능한 값:

* true
* 거짓

 a| 
부울



 a| 
제안된 클러스터 오류
 a| 
제안된 스토리지 노드를 사용하여 스토리지 클러스터를 생성하면 발생할 수 있는 오류입니다.
 a| 
문자열 배열

|===


== 요청 예시

이 방법에 대한 요청은 다음 예와 유사합니다.

[listing]
----
{
	"method": "CheckProposedCluster",
	"params": {
	"nodes": [
		"192.168.1.11",
		"192.168.1.12",
		"192.168.1.13",
		"192.168.1.14"
	]
},
	"id": 1
}
----


== 응답 예시

이 메서드는 다음 예와 유사한 응답을 반환합니다.

[listing]
----
{
	"id": 1,
	"result": {
		"proposedClusterValid": true,
		"proposedClusterErrors": [ ]
	}
}
----
